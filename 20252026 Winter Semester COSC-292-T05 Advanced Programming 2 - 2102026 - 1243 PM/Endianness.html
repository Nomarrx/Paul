Lesson — Endianness

Learning Outcome(s): LO5 — Develop code to reference and manipulate static memory with pointers.

Assessment alignment: Midterm 1 (Memory concepts).

Endianness

There are two common schemes for arranging the bytes in a variable:

  • Big-endian: most significant byte at the smallest memory address
  • Little-endian: least significant byte at the smallest memory address

The least significant byte is the byte that changes the value the least, while the most significant byte changes the value the most.

For instance, given int x = 999999999;, in hexadecimal it is 3B9AC9FF. 3B is the most significant byte, while FF is the least significant byte.

Impact of changing bytes

  • Changing the most significant byte to 009AC9FF results in a decimal value of 10,144,255 (over 980 million different from the original value).
  • Changing the least significant byte to 3B9AC900 results in a decimal value of 999,999,744 (only 255 different from the original value).

Memory representation

On a big-endian machine, the value might be represented as:

Variable Memory address (going up) Value (in hex)
0x0012FF83 FF
0x0012FF82 C9
0x0012FF81 9A
int x 0x0012FF80 3B

On a little-endian machine, the value would be represented as:

Variable Memory address (going up) Value (in hex)
0x0012FF83 3B
0x0012FF82 9A
0x0012FF81 C9
int x 0x0012FF80 FF

The platforms we are working on (both x64 and x86) are little-endian machines. Therefore, any further descriptions of memory that we use will be little-endian.

See Wikipedia: Endianness for more details.

Example in little-endian notation

Returning to our previous example, iTest would be written as the following in little-endian notation:

Byte’s Address Alias Byte’s Value
0x0019FEDF ?
0x0019FEDE ?
0x0019FEDD ?
0x0019FEDC cTest 0x41 = 'A'
0x0019FEDB 0x12
0x0019FEDA 0x56
0x0019FED9 0x9A
0x0019FED8 iTest 0xDE

By writing the memory addresses going up (lowest at bottom to highest at top), we can read the value of iTest going down: 0x12569ADE.

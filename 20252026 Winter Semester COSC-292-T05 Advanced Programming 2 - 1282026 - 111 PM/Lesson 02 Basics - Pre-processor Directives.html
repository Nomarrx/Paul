Lesson 02 Basics - Pre-processor Directives

Pre-processor Directives

Quick highlights:

  • Starts with # (e.g., #include, #define)
  • Runs before compilation; performs substitution

Details

C also includes pre-processor directives. Before the program is compiled, the pre-processor reads the source file looking for directives (starting with #). The pre-processor then applies the directives to the code.


#include is a pre-processor directive that says that there is some existing code that we want to use. The existing code is then substituted into the source code file. It can include definitions of functions that are in libraries that can be compiled into the final executable. Files referenced by #include can be included in <> (for files in the built-in standard libraries) or "" for local project files. #include is similar to “Using” statements in C# or “import” in Java.

Examples include:


#include <stdio.h> // add the standard input/output library
#include <stdlib.h> // add the standard library
#include "myfile.h" // add the local header file myfile.h

Another type of pre-processor directive is the define statement. The simplest version of the define statement is a substitution of a name (preferably meaningful) for a value, such as the following. Note that there is no semicolon at the end of the line – #define is not a programming language instruction.

#define PI 3.141519

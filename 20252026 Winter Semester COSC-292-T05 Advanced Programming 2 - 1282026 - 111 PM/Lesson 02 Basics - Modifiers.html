Lesson 02 Basics - Modifiers

Modifiers

Quick highlights:

  • short, long, signed, unsigned
  • long long available since C99

Details

C data types can also be altered by using modifiers in front of the type. The modifiers are short, long, unsigned, and signed. They are used as follows:


short can be used in front of int (or as a shortcut, by itself as just short). A short int is guaranteed to be at least 16 bits or 2 bytes but could be more.
long can be used in front of int or double. The shortcut long by itself stands for long int. A long is guaranteed to be at least 32 bits or 4 bytes. Note that this differs from Java, where a long is defined as always being 64 bits or 8 bytes.

Since C99, you can also use a long long int (or just long long) which requires at least 64 bits (8 bytes).

unsigned and signed can be used in front of char and any variation of int. Signed variables can contain both positive and negative values, while unsigned variables can only contain nonnegative values (and thus have twice the range of positive numbers). For instance, unsigned types are used in the definition of bitmap image files, since things like the number of colors and file size can only be nonnegative. The variations of int are all signed by default unless the unsigned modifier is used. It is system dependent whether char by itself is signed or unsigned, so use the modifier in any cases where you need to be sure. For instance, signed char can contain values from -128 to +127, while unsigned char values range from 0 to 255.

Let’s write some code to try using modifiers. In main, add a call to the function modifiers:

modifiers();

Then add the function modifiers as follows, again above main:


void modifiers()
{

// Note that C defaults to signed values (can be + or -)

// except for char, which is system-dependent

unsigned char u = 255; // unsigned char can store values from 0 to 255

signed char s = 127; // signed char can store values from -128 to 127

// "short" and "long"


	short int x; // will be 2 bytes on this platform
	short y; // short form for writing a short int
	long int z; // no effect on this platform

// Note that the only guarantees are that short <= int <= long

// and that short must be at least 16 bits and long must be at least 32 bits


	long long big; // since C99; 8 bytes on this platform

	long float f; // nonstandard - doubles the size to 8 bytes (same as double)
	long double d; // no effect on this platform (same as double)

// Try printing e and c without the next two lines and then with them

// Add 1 to e and c (to see the effect on unsigned vs. signed)

u = u + 1;

s += 1;


	printf("The value of u is %d\n", u);
	printf("The value of s is %d\n", s);

	printf("Size of short: %zu %zu\n", sizeof(x), sizeof(y));
	printf("Size of long: %zu\n", sizeof(z));
	printf("Size of long long: %zu\n", sizeof(big));
	printf("Size of long float: %zu\n", sizeof(f));
	printf("Size of long double: %zu\n", sizeof(d));
}

Procedural languages

The language that we will be studying in this class is C. C is a procedural programming language. A procedural language is one in which the tasks of a program are broken into procedures (also called subroutines or functions). To write a program, you decide which procedures and data structures you want to use and choose the best algorithm for each procedure. Support is based on functions communicating by passing arguments and returning values. Examples of procedural languages include C, Fortran, COBOL, BASIC, and Pascal.

This can be compared to object-oriented programming, where you decide what classes you need, provide a full set of operations for the class, and make commonality explicit using inheritance. You also separate what can be accessed from inside and outside the class using modifiers like private and public. Examples of object-oriented languages include Java and C#. Note that when you started writing programs in Java using only a single class and static methods, you were essentially doing procedural programming.

See also https://en.wikipedia.org/wiki/Procedural_programming

C is a popular language (among the top four in the TIOBE index - https://en.wikipedia.org/wiki/TIOBE_index - and along with C++, in the top 5 of the PYPL Popularity of Programming Language Index - https://pypl.github.io/PYPL.html). It is known for its low-level capabilities which allow it to create efficient programs that take advantage of the design of the machine that it is working on, making it well suited for things like operating systems and device drivers. Thus, part of what you will learn in this class is how the machine deals directly with data, seeing things that other languages may have hidden.

The name C came from its predecessors, BCPL and then B, with C being the next letter in line. C was developed in the early 1970s and was used to write the UNIX operating system. C did not have an original standard, although the book by Brian Kernighan and C’s creator Dennis Ritchie was commonly used, called K&R C from the initials of the authors. C was later standardized by the American National Standards Institute (ANSI) and the International Organization for Standardization (ISO), with versions coming out in 1989/90 (C89 or C90), 1999 (C99), and later.

See also https://en.wikipedia.org/wiki/C_(programming_language)#History

C++ is an object-oriented superset of C, which means that it includes all the features of C but also includes extra object-oriented features. It was developed in the early 1980s. We will not cover C++ in this class, but you will have a basic background in it based on covering C.

See also https://en.wikipedia.org/wiki/C%2B%2B

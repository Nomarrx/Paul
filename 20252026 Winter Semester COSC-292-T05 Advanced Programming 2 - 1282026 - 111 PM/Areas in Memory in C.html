Lesson — Areas in Memory in C

Learning Outcome(s): LO5 — Understand and manipulate memory areas in C.

Assessment alignment: Midterm 1 (Memory concepts).

Areas in Memory in C

C programs use several distinct memory areas:

Text Area

The text area stores program code, string literals, and global constants. These locations are read-only; attempting to modify them will crash your program in Visual Studio. For example, the address of the memoryMap function is far from local variables because it resides in the text area.

Four Main RAM Areas for Variables

  1. Data Area: Global variables initialized to a value other than 0.
  2. BSS Segment: Global variables not initialized (defaults to 0).
  3. Heap: Dynamically allocated variables at runtime (e.g., via malloc in C, or new in C#/Java).
  4. Stack: Variables and parameters declared within functions (stack frame).

Why two areas for globals? Before main() runs, the C runtime sets all bytes in the BSS segment to zero (fast bulk initialization). For the data area, it sets each variable to its initial value individually.

Example Program

#include <stdio.h>

int globalVar;
double pi = 3.14159;

int main()
{
    char c;
}

Review: Where are these stored?

  • globalVar — BSS segment (global, not initialized → defaults to 0)
  • pi — Data area (global, initialized to non-zero)
  • c — Stack (local variable in a function)

Static variables

There is a special kind of variable called a static variable, but this is different from global variables defined with the static keyword. A static variable is defined in a function, is only known within that function, but exists for the lifetime of the program. Rather than being allocated on the stack, memory for static variables is allocated either in the BSS segment or the data area (same as global variables). They work just like global variables, but their scope is limited to the function that they are declared in.

It is good programming practice to limit the scope of variables; when you use them, static variables are better to use than global variables.
